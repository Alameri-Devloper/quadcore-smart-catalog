# Workspace Boundary | حدود مساحة العمل

**Status:** Foundation · **Last Updated:** 2026-07-19 · **Scope:** Workspace domain

## English

Workspace belongs to a Company and is the ownership boundary for configuration and future Workspace-scoped repository operations. V1 is Single-Workspace and Multi-Branch: trusted Application context supplies the one active Workspace automatically, with no end-user selector or operational tenant administration. V2 introduces Multi-Workspace and Multi-Tenant operation. Branch remains distinct from Workspace; Branch and Warehouse quantities belong to Inventory. Configuration such as `autoPublishReadyProducts` belongs here, though this foundation implements no configuration behavior.

## العربية

تتبع Workspace شركة وتمثل حد الملكية للتهيئة وعمليات المستودعات المقيدة بمساحة العمل مستقبلاً. الإصدار V1 هو Single-Workspace وMulti-Branch؛ يمرر سياق Application الموثوق مساحة العمل النشطة الوحيدة تلقائياً دون محدد للمستخدم أو إدارة تشغيلية للمستأجرين. يقدم V2 تشغيل Multi-Workspace وMulti-Tenant. يبقى Branch مختلفاً عن Workspace، وتنتمي كميات الفروع والمستودعات إلى Inventory. تنتمي إعدادات مثل `autoPublishReadyProducts` إلى هذا المجال، ولا ينفذ هذا الأساس سلوكها.
