# Product Entry Architecture | معمارية إدخال المنتج

**Status:** Active foundation · **Last Updated:** 2026-07-19 · **Scope:** Product-entry workflow

## English

Product Entry is a guided, Responsive First workflow for Mobile, Tablet, and Desktop as equal first-class environments. It must support touch, mouse, and keyboard interaction without treating Desktop or Tablet as secondary. Components collect input; services coordinate business behavior; repositories provide data. It preserves state, uses context to avoid repeated questions, validates through domain rules, explains recovery, and converges on one Save Product action. Standard, context, batch, Draft, Excel, and AI-assisted modes have different implementation status and must be checked in the legacy design documents and capability map.

## العربية

إدخال المنتج مسار موجه يتبع Responsive First للجوال والجهاز اللوحي والكمبيوتر كبيئات أساسية ومتساوية. يجب أن يدعم اللمس والفأرة ولوحة المفاتيح دون اعتبار الكمبيوتر أو الجهاز اللوحي ثانوياً. تجمع المكونات المدخلات، وتنسق الخدمات سلوك العمل، وتوفر المستودعات البيانات. يحفظ المسار الحالة ويستخدم السياق لتجنب التكرار ويتحقق بقواعد المجال ويشرح الاستعادة وينتهي بإجراء حفظ واحد. تختلف حالة تنفيذ طرق الإدخال ويجب الرجوع إلى وثائق التصميم السابقة وخريطة القدرات.
