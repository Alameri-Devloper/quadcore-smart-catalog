# Documentation Governance | حوكمة التوثيق

**Status:** Accepted · **Last Updated:** 2026-07-19 · **Scope:** Documentation lifecycle

## English

Every important decision is documented in English and Arabic. Superseded documents are archived. ADRs are immutable historical records except status changes and explicit corrections. New domains require boundary documentation. Roadmap ideas are never presented as implemented architecture. Documentation review is part of Definition of Done, and related documents use relative links. UX, implementation, test, and acceptance documentation must use Responsive First language and explicitly cover Mobile, Tablet, Desktop, touch, mouse, and keyboard verification. Documentation must not describe Desktop or Tablet as secondary environments.

## العربية

يوثق كل قرار مهم بالإنجليزية والعربية. تحفظ الوثائق المستبدلة في الأرشيف. سجلات ADR تاريخ ثابت لا يعدل إلا لتغيير الحالة أو تصحيح معلن. يتطلب كل مجال جديد توثيق حدوده، ولا تعرض أفكار الخارطة كأنها منفذة. مراجعة التوثيق جزء من تعريف الاكتمال، وتستخدم الروابط النسبية. يجب أن تستخدم وثائق UX والتنفيذ والاختبار ومعايير القبول مصطلح Responsive First وأن تغطي صراحة الجوال والجهاز اللوحي والكمبيوتر والتفاعل باللمس والفأرة ولوحة المفاتيح، وألا تصف الكمبيوتر أو الجهاز اللوحي كبيئة ثانوية.
