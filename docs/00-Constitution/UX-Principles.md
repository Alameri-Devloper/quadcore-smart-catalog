# UX Principles | مبادئ تجربة المستخدم

**Status:** Accepted · **Last Updated:** 2026-07-19 · **Scope:** UX constitution

## English

Zero Training, one primary action, smart defaults, Never Ask Twice, Never Lose User Work, progressive disclosure, invisible lifecycle, Responsive First, accessible guidance, and clear recovery behavior. Mobile, Tablet, and Desktop are equally first-class environments. Each experience must be purpose-designed for its available space and must support touch, mouse, and keyboard interaction where applicable. Desktop and Tablet must never be treated as secondary or as stretched Mobile layouts. A new employee should understand the main workflow in about ten minutes. Technical complexity stays invisible and the system teaches itself.

## العربية

تقوم التجربة على عدم الحاجة إلى تدريب، وإجراء رئيسي واحد، وافتراضات ذكية، وعدم طلب المعلومة مرتين، وعدم فقد عمل المستخدم، والإظهار التدريجي، وإخفاء تعقيد دورة الحياة، ومبدأ Responsive First، والإرشاد المتاح، ومسارات استعادة واضحة. الجوال والجهاز اللوحي والكمبيوتر بيئات أساسية ومتساوية، ويجب تصميم تجربة مناسبة لمساحة كل منها ودعم التفاعل باللمس والفأرة ولوحة المفاتيح بحسب السياق. لا يجوز اعتبار الكمبيوتر أو الجهاز اللوحي نسخة ثانوية أو مجرد تمديد لتصميم الجوال. ينبغي أن يفهم الموظف الجديد المسار الأساسي خلال نحو عشر دقائق وأن يشرح النظام نفسه بنفسه.
